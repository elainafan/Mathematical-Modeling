# Q2：随机节点故障下的交通网络健壮性 建模思路与算法设计

## 1. 核心概念与数学定义

本题旨在评估面对“随机干扰（如散发性地质灾害、随机断电）”时，8个城市交通网络的抵抗能力。

*   **自变量（破坏比例 $q$）**：
    $$q = \frac{N_{removed}}{N_{total}}$$
    其中，$N_{removed}$ 为当前已经瘫痪的节点数，$N_{total}$ 为网络最初的节点总数。
    
*   **因变量（网络性能 $P(q)$）**：
    题目定义为“当前网络最大连通分量与原来网络规模的比值”。
    $$P(q) = \frac{N_{LCC}(q)}{N_{total}}$$
    其中，$N_{LCC}(q)$ 表示在破坏比例为 $q$ 时，剩下的图里**节点数最多的那个连通块（Largest Connected Component, LCC）**所包含的节点数量。

*   **网络健壮性（$R$）**：
    题目要求定义为性能曲线的积分。在离散计算中，我们使用梯形法则或黎曼和近似计算：
    $$R = \int_{0}^{1} P(q) dq \approx \sum_{i=1}^{m} P(q_i) \Delta q$$

## 2. 核心难点与解决策略

### 2.1 消除随机误差：蒙特卡洛（Monte Carlo）模拟
“随机故障”意味着每次瘫痪的节点顺序都是不同的。如果只随机生成一种破坏顺序，得出的曲线会有极大的偶然性（锯齿状波动）。
**对策**：采用蒙特卡洛模拟。对每个城市，独立进行 $M$ 次（建议 30-50 次）完整的模拟实验。最后将这 $M$ 次在相同 $q$ 下的 $P(q)$ 求平均值，得到一条平滑的、能反映统计规律的期望性能曲线 $\bar{P}(q)$。

### 2.2 降低时间复杂度：分区间步长采样
节点数高达几万（如成都18300个），如果每次删除1个节点就计算一次全图连通块，单个城市需计算 18300 次，50次蒙特卡洛就是将近 100万次，时间上不可接受。
**对策**：采取“分批次破坏”。例如设置步长 $\Delta q = 0.01$ (1%)。即每次瘫痪 1% 的节点（约180个），计算一次 $P(q)$，共计算100次即可画出完整曲线。

### 2.3 寻找“显著变化点”（网络崩溃临界点）
题目要求回答：“当比例达到多少时，健壮性发生显著变化”。
在网络科学（渗透理论 Percolation Theory）中，这对应着网络从“整体连通态”破碎为“无数个小孤岛”的**相变阈值（Percolation Threshold）**。
**对策**：在数学上，“显著变化”即体现为曲线下降最猛烈的地方。我们计算性能曲线的一阶导数（差分）：
$$ \Delta P(q_i) = P(q_i) - P(q_{i-1}) $$
使得 $|\Delta P(q)|$ 绝对值最大的那个 $q$ 点（也是二阶导数过零点的位置），就是网络崩溃的显著变化比例。

## 3. 算法执行流程

针对 8 个城市的 JSON 文件，分别独立执行以下流程：

1. **图的初始化**：读取 JSON，运用 `networkx` 库构建图 `G_init`。记录初始节点总数 $N_{total}$。
2. **初始化存储**：准备一个数组记录各比例 $q \in [0, 0.01, 0.02, ..., 1.0]$ 对应的网络性能。
3. **蒙特卡洛主循环 (例如进行 30 次)**：
   * 克隆出当前模拟用的图 `G_current = G_init.copy()`。
   * 获取所有节点的 ID 列表，并使用 `random.shuffle()` 将其**完全随机打乱**，生成破坏序列。
   * 按 1% 的批次，从序列中取出节点，将其从 `G_current` 中 `remove_nodes`。
   * 调用 `nx.connected_components()` 找出最大连通分量，计算 $P(q)$ 并保存。
4. **统计与计算**：
   * 将 30 次实验的曲线取平均。
   * 计算曲线下方的面积（健壮性 $R$）。
   * 对平均曲线求一阶差分，找出差分绝对值的最大值所在的 $q$ 作为“显著变化阈值”。
5. **结果输出**：
   * 绘制 8 个城市的抗毁性平滑对比折线图。
   * 输出 CSV 表格，包含：城市名、健壮性积分 $R$、显著发生变化的破坏比例 $q_{critical}$。

## 4. 模型进阶与优化：基于双连通核的结构健壮性指标 ($Q_{bc}$)

在基础实现（Baseline）中，我们仅通过最大连通分量 (LCC) 的规模来定义网络性能。但在实际的城市路网中，这种定义存在**巨大的“伪连通”漏洞**：两个庞大区域即便只靠一条跨江大桥（桥边）或一个关键路口（割点）相连，LCC 依然会视其为一个整体，但这在交通抗毁性上是极其脆弱的。

为此，本团队引入了**边双连通分量 ($B_E$)** 和 **点双连通分量 ($B_V$)** 来补充刻画网络内部的连接冗余性。
*   **边双连通**：网络中任意两点间至少有两条不相交的路径（即不存在“一断就全断”的桥边）。
*   **点双连通**：网络中不存在单一的割点瓶颈。

我们定义瞬时结构性能为：
$$ Q_{bc}(f) = \beta \frac{|LCC(G_f)|}{N} + (1-\beta)\left(\alpha \frac{|B_E(G_f)|}{N} + (1-\alpha) \frac{|B_V(G_f)|}{N}\right) $$
其中 $\alpha, \beta \in [0, 1]$ 为权重参数（本实验中取 $\alpha=0.5, \beta=0.5$）。
新的累计结构健壮性积分定义为：
$$ R_{bc} = \int_{0}^{1} Q_{bc}(f) df $$

### 4.1 算法工程与性能优化 ( Engineering Optimization )
双连通核的计算复杂度 ($O(V+E)$) 远高于普通连通块，且附加蒙特卡洛多重采样后，如果采用常规的多进程 (`ProcessPoolExecutor`) 往往会导致 Windows IPC (进程间通信) 管道被海量的序列化图对象（Pickle）撑爆，引发内存死锁（Deadlock）。

**解决对策**：
1. **彻底解耦**：将核心仿真模块 (`q2_compute_bc.py`) 与 积分/绘图模块 (`q2_plot_bc.py`) 分离，持久化中间仿真数据，避免重复计算。
2. **共享内存并发**：抛弃多进程并发，改用**多线程池 (`ThreadPoolExecutor`)**。由于 Python 字典和 NetworkX 视图在并发读取时的安全性，各线程仅向主进程“只读借用”图拓扑 `G`，并在本地生成私有子图副本 `G_temp` 进行破坏和算分。这使得内存占用始终维持在最低水位，算力（CPU）得以 100% 释放，成功支撑了 8 个百万级算力的城市路网 $Q_{bc}$ 健壮度重测。
